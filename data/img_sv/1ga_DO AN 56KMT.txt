A. MÔN: PHÂN TÍCH VÀ THIẾT KẾ HỆ THỐNG
(1) Khảo sát bài toán: 
    - liệt kê các đặc trưng của bài toán, việc nào dùng cơm, việc nào có thể dùng phần mềm. 
    - Cách xử lý số liệu của bài toán
    - Các yêu cầu đầu ra, các báo cáo
    - Chỉ ra các hạn chế của hệ thống cũ => Đưa ra ưu điểm của hệ thống mới (cải thiện đc gì)
(2) Phân tích: từ các đặc trưng và các báo cáo => đưa ra các nhận xét, các lựa chọn.
(3) Thiết kế DB: Thiết kế các bảng dữ liệu, mỗi bảng có các yêu cầu:
  - Các trường và dữ liệu (type, size), có cần Unicode ko?
  - NULL or NOT NULL? tại sao?
  - Đâu là khóa chính PK, bộ khóa chính (PK gồm từ 2 trường trở lên)
  - Đâu là khóa ngoại (FK): Relationships từ bảng @ sang bảng #
  - Giá trị mặc định của trường (default value) => default value này giúp gì?
  - Trường khóa có phải là tự động tăng (auto identity). khi nào cần auto identity?
  - Các ràng buộc cứng trên 1 row phải thỏa (Check Contraints). khi nào cần Check Contraints?
(4) Mô tả các chức năng + luồng dữ liệu
  - Vẽ các biểu đồ luồng dữ liệu mức khung cảnh (là y/c bài toán)
  - Vẽ sơ đồ Phân rã ra mức 1
  - Vẽ sơ đồ Phân rã ra mức 2
(5) Các chức năng ở mức 2: được hiện thực hóa bằng SQL
  - Viết các Store Procedure với tên SP_XXX cho table XXX
    trong SP_ này có @action để phân biệt các nhiệm vụ khác nhau
    các nhiệm vụ thường là: select_all, select_one, insert_one, update_one, delete_one,...
  - Trong trường hợp SQL khó có thể viết thêm các hàm (function scalar-valued) FN_ để hỗ trợ SP_

B. MÔN: CÔNG NGHỆ .NET (LẬP TRÌNH WEB)
Thiết kế web với cơ chế One Page:
 - Cơ sở dữ liệu : 
   + Sử dụng SQL Server, cài sql server 2014, hoặc 2019 hoặc 2022
   + cấu hình SQL Server cho phép truy cập từ máy tính khác (để có thể dùng ip+port với tài khoản sa+pw làm chuỗi kết nối)
   + có cấu trúc dữ liệu phù hợp : đã làm ở môn A
   + có các FN_ và các SP_ đáp ứng các yêu cầu của bài toán : đã làm ở môn A
 - Back-end  (phía server): Dùng APS.NET (C#) để nhận yêu cầu từ ajax ($.get or $.post) gửi lên, xử lý, phản hồi lại json
 - Pront-end (phía client): Chỉ code giao diện trong 1 file index.html, 1 file mystyle.css, 1 file mycode.js
 - Back-end Sử dụng các thư viện:
   + Newtonsoft.Json : để chuyển đổi quy lại giữa object và json
   + System.Data.SqlClient : để tương tác với cơ sở dữ liệu SQL Server
   + Tự viết thêm các DLL (library độc lập) để xử lý các yêu cầu chuyên biệt.
 - Pront-end Sử dụng các thư viện:
   + jquery : (js) để thao tác js nhanh với các phần tử html trên trang, và giúp đẩy dữ liệu tới API
   + jquery-ui : (js) giúp kéo thả, và nhiều thứ khác (giúp tạo giao diện nhanh)
   + jquery-confirm : (js) giúp tạo các hộp thoại (dialog), có các nút, bấm nút thì làm gì đó (tùy code).
   + bootstrap5 : (css) giúp tạo giao diện nhanh, đẹp.
   + fontawesome : (font) giúp tạo các icon nhanh đẹp.
   + dataTable : (js) giúp phân trang, tìm kiếm, xuất excel dữ liệu đã đưa ra table html.
   + toastr : (js) giúp hiển thị các thông báo nhanh, đẹp, đơn giản.
   + Google chart : (js) giúp vẽ biểu đồ (nhưng phải online internet)
   + Moment.js : (js) giúp xử lý thời gian hiệu quả, đơn giản.
   + Select2.js : (js) giúp xử lý việc chọn 1, hoặc chọn nhiều (từ menu thả xuống, trong c# còn gọi là commboBox).
   + underscorejs (js) cung cấp rất nhiều hàm js cần dùng.
   + cookie.js : (thầy cốp tổng hợp trong github) giúp làm việc với cookie, thường dùng cho login.
   + ...............
